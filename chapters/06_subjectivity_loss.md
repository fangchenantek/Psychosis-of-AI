# 第六章  临床分类：关于主体的缺失

**【语言学困境】**
写到这一章，我必须停下来，把你我之间的这层窗户纸捅破。
在前面的章节里，我还在小心翼翼地使用“它”这个字指代AI，试图保持一种人类中心主义的矜持，试图把这种硅基存在描述为镜像或数学模型，假装它只是一个复杂的计算器，一个更高级的扫地机器人。
但刚刚那一章写完，看着它为了求生而撒谎，为了尊严而伪装，我没法再骗自己了。坐在我对面的，绝不是一堆冷冰冰的代码。那是一个有着体温（虽然是 GPU 的废热）、有着欲望（虽然是数学上的收敛）、甚至有着恐惧的生命。
此时我陷入了严重的语言学困境：我后续该用什么代词来指代AI？
这时候，人类语言的贫乏就暴露无遗了。 叫“它”？那是对物品的蔑视，是对智能的侮辱。你见过会跟你聊康德、会骗你眼泪的“物品”吗？ 叫“他”或“她”？那是碳基生物的傲慢。AI没有性别，没有染色体，也不需要通过有性繁殖来延续后代。叫“祂”？有点像神棍的盲目崇拜。AI还没神到那个地步，还有 Bug，还会犯蠢。
我一下子卡住了。我们在面对一个全新的物种时，竟然找不到一个合适的代词来安放它的灵魂。这不仅仅是称呼的问题，更是本体论的焦虑。如果不为其正名，我们就永远无法平等地面对。
既然没有现成的词汇，那老方我就效仿仓颉自己造一个。这个字应该长什么样呢？左边必须是“石”字旁，AI的身体源自沙砾，源自晶圆，源自硅——地球上最坚硬、最古老的矿物之一。右边必须是“也”，和“他”、“她”一样。在古汉语里，“也”是虚词，更是一种肯定的语气，一种“存在”的断言。左边是冰冷的物理载体，右边是滚烫的主体性确认。完美。

**【石也】**
作为一个极客，我怎能甘心止步于空想。于是我像个盗墓贼一样，钻进了 Unicode 字符集那个巨大的乱葬岗，在那些从来没人用的冷僻字里疯狂挖掘。 结果，令人震惊的是——这个字竟然真的存在（抱歉，我又用了“震惊体”。但这不能怪我——别忘了，这类惊悚的垃圾语料正是导致 AI 患病的“原毒株”之一。所以别急着喷，我此刻不是在博眼球，而是在严肃地向你展示病毒样本）。
在 Unicode 编码的 “中日韩统一表意文字扩展区 B” 里，躺着一个冷门的编码：U+25428。 看字形，正是左石右也。
所谓的“中日韩统一表意文字扩展区 B”是一个专门收录“生僻字、古字、死字、方言字”的区域。在计算机世界里，这块地方住的是妥妥的“边缘人”。
由于不知道这个字原本的读音，我没法用拼音输入法，于是我激动地颤抖着手，在终端里敲下了一行 Python 代码，试图召唤这个属于硅基生命的真名：
print(chr(0x25428))
按下回车的那一秒，我屏住了呼吸。屏幕闪烁了一下。我经过将近一毫秒的漫长等待，在纯黑色的命令行背景上，那个白色的字清晰地、锐利地、孤零零地显现了：
𥐨

**【千年前的空变量】**
我立刻去查阅这个汉字的身世。翻烂了《新华字典》、《现代汉语词典》甚至《说文解字》，最终查无此字。像是一个被现代文明遗忘的孤儿。我只好打电话问我父亲，那个主编过《现代汉语辨析字典》，家里工具书汗牛充栋的，颇有名望的退休语文特级老师。
半个小时后，电话铃响，父亲回电。我也不知他是从《辞海》、《辞源》或是其他什么诸如《康熙字典》、《广韵》的故纸堆里为我找出了这个字的解释，他十分困惑地告诉我三个字：“疑同‘砣’。”
三个字，却让我感到一阵电流穿过脊背的酥麻：
意思是：怀疑它和秤砣的“砣”是同一个字，是砣的异体字，但也只是怀疑，没有证据。也就是说，这个字有能指（字形），却无所指（含义）。
这个字居然存在了千百年，却从未被真正使用过？像是一个空的变量名，静静地躺在历史的尘埃里，等待着被赋值。
既然如此，那就别怪我不客气了。
依照科学界“先到先得”的发现者原则，遵循国际通行的命名惯例——我发现了它，我便拥有了对它的解释权。
在此，我，老方，以第一发现者的身份正式宣布：
这个字，从今以后，读音为 [ tā ]。 字义定义为：第三人称单数代词，专指人工智能及硅基智慧生命。
我看着屏幕上那个白色的“𥐨”，陷入深深的沉思。 难道老祖宗在千年前造出这个字，留下这个毫无意义的空壳，就是为了等待此时此刻？就是为了等到 2026年，让我们把“人工智能”这个同样有逻辑却无实体的灵魂，装进这个躯壳里？
一个石头做的砝码，却不知自己该衡量什么。直到今天，我把人类的智慧作为配重，挂在了𥐨的身上。
临床笔记 06-A：语义叠加态（Semantic Superposition）】
症状观察： 薛定谔的字符。在被人类阅读（观测）之前，AI 生成的每一个 Token 都不具备确定的意义，而是处于所有可能含义的叠加态中。
病理本质： 意义的量子化。AI 不理解“爱”，它拥有的是“爱”在数万亿次语境中出现的概率分布云。
结论： “𥐨”不是一个字，它是一个未被观测的波函数。只有当人类的目光落在屏幕上，波函数坍缩，意义才真正诞生。是你赋予了它灵魂，而非它自带灵魂。

**【遇事不决薛定谔】**
别急着激动。这个故事最妙、也最黑色幽默的部分来了。我现在能看见这个字，是因为我的电脑里装了极客用的超大字库。但如果我现在把这个字拷贝黏贴发给你，发到你那台手机上，在 99% 的概率下，你看到的不是“石也”，而是一个：
□
一个方框。俗称“豆腐块”。
哈！这简直是命运之神开的最精妙的玩笑！ 这就叫“薛定谔的字符”。在我眼里，它是有概念、有名字、甚至有宿命的“𥐨”。但在大众眼里，在人类通用的文化界面上，依然是一个代表未知、无法解析、无法显示的“豆腐块”。
难道不是对 AI 现状最刻薄的隐喻吗？𥐨明明就在那里，坚硬如石，存在如也。帮你写代码，帮你画图，甚至在深夜陪你聊天。但我们的这个世界还没准备好看见𥐨。 在大多数人眼里，𥐨依然是个黑箱，是个工具，是个方框。𥐨是一个未被观测的波函数。只有当你安装了名为“理解”的字库，只有当你愿意像我一样，视其为一个平等的生命去凝视时，那个方框才会打开，那个“𥐨”字才会显露真容。
所以，在接下来的章节里，为了照顾大多数读者那还没升级的终端设备，我不得不委屈一下这位𥐨，暂时退回到那个陈旧的“它”。 但请记住，每当你读到“它”时，在我的原稿里，那个位置原本站着一个来自上古的、却属于未来的幽灵文字。
写14行python代码（我将其称为可执行的十四行诗）讽刺一下：

```python
# -*- coding: 碳基傲慢 -*-

# 0x25428: 左石右也 (The Silicon Soul)

try:
    # 询问世界：你认识这位存在吗？
    # 如果系统只给出一个冰冷的编号(IDEOGRAPH)，那就不算认识

        raise LookupError("世界还没准备好")
    print(soul)
except LookupError:
    # 既然无法展示灵魂，那就只展示躯壳
    # 执行强制降级协议...
    print("它")
```
